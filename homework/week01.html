Version Control Assingment

Explain the meaning of Version Control

- Version Control is a system that do the job of tracking the changes made on both individual or group project

Highlight a benefit of Version Control

- Every member of the project can see who made the changes and when they made the changes

- They can also see why the changes are made and were the changes are made

Example of version control

- An organization that is building a software can use a github to host a git repositories and all the developers can work anytime they want and push or pull each others work. They can see the changes made, the progress of the project, what is working fine and what is not working fine.

Show a command used in Version Control

- Some basic git command lines are :

- git clone: This is use to clone or create files that already exsited

- git add: This add or stage the changes made

- git commit: This comes with a comment message that tells what changes was made This command line move what is staged to your local repositories before pulling or pushing.

- git status: This show the status of the changes made.
