Encapsulation

Explain the meaning of Encapsulation

- Encapsulation is the act of enclosing your code basically to prevent unwanted access. You can say that it is a principle of keeping some code private mostly code that will likly not change easily.

Highlight a benefit of Encapsulation

- Encapsulation help to reduce the complexity of a code

- Encapsulation Makes it easier for programmers to maintain code and avoide too many changes when developing a program.

- It helps programmers to reuse a code or function instead of rewriting a code over and over again.

- Encapsulation helps keep codes or method private, so other class will not access them without a getter or an allowed source.
